# 変更履歴

バージョンは `1.0` → `1.1` → `1.2` ... の順で、アップデートのたびに小数点以下を1つずつ上げる形式とする(軽微な修正のみの場合は例外的にパッチ番号を付けることがある。例: v1.1.1)。

## v1.2 (2026-07-19)

試合20260719(vsレッドリボン)の事後レビューと既存バックログから確定した8件の修正・機能追加。

- 本塁打選択時は「生還した」チェックを自動でON・変更不可にし、チェック漏れを防止(ラベルに自動注記を表示)
- 失策(エラー)の選択肢をゴロ/フライの区別なしに簡素化し、投手・捕手を含む全9ポジション分に拡大
- 得点圏打数・得点圏安打を自動計算(`aggregate.py`。打席開始時点の2・3塁走者の有無で判定)
- 練習試合/公式戦の区別を追加。練習試合はスカイツリーグ登録の対象外として自動スキップ(TeamSへの登録は従来通り)
- エラー・振り逃げ出塁・単打・野選で、打者走者自身の追加進塁(悪送球等)を記録できるように
- 助っ人選手をその場で登録可能に(表示名のみ入力、IDは自動生成)。「その他(自由入力)」を選んでも記録が飛ばされていた不具合を修正
- 守備位置の交代ログを`lineup_history`テーブルに記録(将来の守備成績算出用)
- 打席の編集機能を強化し、ランナーの動き(生還・進塁・アウト)も編集可能に。冪等性ガード(オフライン再送での二重反映防止)・編集時点の走者状態の再現・後続打席から参照されている打席の保護を実装



- 二塁打の走者進塁デフォルトが「塁の詰まり」を考慮していなかった不具合を修正。一・二塁に走者がいる状態で二塁打が出た場合、二塁走者は一塁走者(三塁へ進む想定)と衝突するため、二塁走者のデフォルトを「三塁」ではなく「生還」にする(一塁が空いている場合は従来通り「三塁」のまま)

## v1.1.2 (2026-07-14)

- アウトカウントの直後に短い塁状況表記(「一塁」「一三塁」「満塁」等)を復活
- スコア表示の「(参考、公式記録は別途手動確認)」の注釈を、スコア本体より小さい文字に分離

## v1.1.1 (2026-07-14)

v1.1直後の実運用フィードバックを反映した軽微な修正。

- 走者進塁デフォルトの精度修正: 二塁走者+二塁打のケースのみ「生還」ではなく「三塁」をデフォルトにする(三塁までは高確率で進めるが生還はクロスプレーになりうるため)。一塁走者+二塁打・三塁打・本塁打の各デフォルトは変更なし
- 打席送信後にページ上部(現在の状況・スコア表示)へ自動スクロール。自分の送信操作のときのみ動作し、他端末からのリアルタイム更新では動作しない
- 走者の文字表記(「ランナー: 一塁田中」)をSVGダイヤモンド表示と同じpointer-box内に統合(独立していた`runners-box`は削除)

## v1.1 (2026-07-14)

- 強制進塁デフォルトの対象拡大: 単打・エラー出塁は四球・死球と同じ押し出し連鎖、二塁打・三塁打・本塁打は塁が埋まっていれば一律2/3/4個先の塁へ進む前提をデフォルト選択にする
- 入力中バッジ: Supabase RealtimeのPresenceで、今この試合のフォームを開いている人の名前を表示
- オフラインキューの永続化: 送信待ち・送信失敗中のリクエストを`localStorage`にも保存し、リロード後も自動で再送を再開する
- 併殺打(GIDP)の自動計算(`aggregate.py`): ゴロアウト+同一プレーでの既存走者アウトを相関させて検出
- 初期画面(`index.html`)に開催中の試合一覧を表示し、`入室する`リンクから直接入室できるようにする(試合開始前にURLを都度配る手間をなくす)
- 検証用試合データ(`opponent_name`に「検証」を含むもの)を作成から24時間後にpg_cronで自動削除
- デザイン改善: スコア表示の強調、攻撃中/守備中を背景色で区別、塁状況をSVGダイヤモンド図で表示、クイックイベントボタンのグループ分け、送信中の状態表示(送信ボタンのテキスト変化)

## v1.0 (2026-07-13)

初回完成版。

- 試合作成・スターティングメンバー登録(`index.html`)、試合中いつでものオーダー編集
- 複数端末からのリアルタイム同時入力(打撃・投手成績・攻守交代)
- 盗塁・盗塁死・走塁死・暴投・ボーク・パスボール・進塁の記録(自チーム・相手チーム両対応)
  - 暴投・パスボールは走者ごとに個別選択、ボークは走者全員が確定で進塁する前提で確認のみ
  - 攻撃中は投手成績記録の設定に関わらず暴投・ボーク・パスボールを使用可能
- 四球・死球での押し出し走者、犠飛(0/1アウト・外野フライ・三塁走者生還)の自動判定・確認導線
- 打点(RBI)の自動計算(生還した走者数から算出。エラー・野選・併殺・三振・振り逃げでは常に0固定)
- 振り逃げ(出塁)の結果選択肢
- イニング表示への塁状況(一塁/一二塁/満塁等)の表示
- 投手成績記録のON/OFF切り替え、OFF時の簡易入力モード
- 送信の自動リトライ・楽観的ローカル反映(Realtime反映待ちによる不整合を防止)
- クイックイベントの競合チェック・二重タップ防止
- 直前の打席のワンタップ取消(他の人の打席と競合した場合は安全に失敗する)
- 試合終了後の集計パイプライン(`import_from_supabase.py`→`validate.py`→`aggregate.py`→`export_excel.py`→`sync_to_gdrive.py`)
